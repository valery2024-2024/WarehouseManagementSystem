# Iteration 2 Plan

## Мета ітерації

Розширити baseline із Lab 34 до повноцінного застосунку з:

- бізнес-логікою;
- persistence layer;
- LINQ-аналітикою;
- патернами проєктування;
- тестуванням;
- розширеним консольним інтерфейсом.

## Сценарії, які переходять у реалізацію

### Реалізовані сценарії

1.Управління товарами

- додавання товарів;
- перегляд товарів;
- пошук товарів;
- сортування та фільтрація.

2.Обробка замовлень

- створення замовлень;
- перевірка залишків;
- автоматичне списання товарів;
- застосування знижок.

3.Аналітика складу

- загальна вартість складу;
- статистика продажів;
- LINQ-запити;
- analytics dashboard.

### Класи з Lab 34, які залишаються без суттєвих змін

- IDataStore<T>

- xUnit test project

### Заплановані точки розширення

#### Strategy Pattern

Для:

- знижок;
- майбутніх типів тарифів;
- акційних механізмів.

#### Factory Pattern

Для:

- створення discount strategies;
- майбутнього розширення типів замовлень.

### Repository Layer

Підготовлений до:

- EF Core;
- SQL Server;
- XML persistence.

### Потенційні ризики

1. Дублювання логіки між service layer та console UI.
2. Складність тестування persistence layer.
3. Можливі проблеми цілісності JSON-файлів.
4. Ускладнення analytics при збільшенні об’єму даних.
5. Потенційні конфлікти при одночасній зміні залишків товарів.

### Основні технічні цілі

- реалізувати JSON persistence;
- реалізувати бізнес-правила;
- додати LINQ-аналітику;

- реалізувати Strategy Pattern;
- розширити тестове покриття;
- підготувати проєкт до Lab 36.
