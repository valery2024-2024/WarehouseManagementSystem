# План ітерації 2

## Сценарії, які переходять у реалізацію

У цій ітерації будуть розширені такі сценарії:

1. Додавання товару на склад.
2. Перегляд товарів.
3. Пошук, фільтрація та аналітика товарів.

## Класи з Lab 34, які залишаються без змін

- Product
- Category
- Supplier
- Warehouse
- StockItem
- Order
- InvalidQuantityException
- InsufficientStockException

## Точки розширення

У проєкті планується використати:

- Repository pattern для роботи з даними;
- Strategy pattern для розрахунку вартості товарів;
- JSON persistence для збереження стану;
- LINQ-запити для пошуку, фільтрації, сортування та аналітики.

## Ризики

- можливе дублювання бізнес-логіки в Console;
- неправильне збереження об’єктів у JSON;
- пошкодження JSON-файлу;
- складність тестування persistence-шару;
- можливість появи слабких інтерфейсів.
