# Release Plan v1.0.0

## Межі релізу

Версія v1.0.0 включає:

* консольний застосунок для управління складом;
* управління товарами;
* обробку замовлень;
* управління складами та залишками;
* управління постачальниками;
* JSON persistence layer;
* реалізацію Repository Pattern;
* реалізацію Strategy Pattern;
* DTO шар;
* unit тести;
* integration тести;
* exception handling;
* CI workflow;
* документацію та UML-діаграми.

## Функціональність релізу

### Основні можливості

* створення та редагування товарів;
* створення та обробка замовлень;
* робота з постачальниками;
* робота зі складами та залишками;
* збереження і завантаження даних у JSON.

### Тестування

* domain tests;
* service tests;
* integration tests;
* exception tests.

### Документація

* README;
* iteration reports;
* test strategy;
* UML diagrams.

## Відкладені можливості

Після завершення курсу можуть бути додані:

* підтримка бази даних;
* авторизація та автентифікація;
* GUI або web-інтерфейс;
* розширена аналітика;
* хмарне розгортання;
* синхронізація в реальному часі.

## Технічний борг

Допустимий технічний борг на момент релізу:

* великий Program.cs;
* обмежене використання async;
* простий консольний UI;
* неповне покриття edge-case сценаріїв.

## Покриття тем курсу

### Повністю використано

* ООП;
* інтерфейси та абстракції;
* generics та колекції;
* LINQ;
* обробка помилок;
* persistence;
* SOLID;
* Strategy Pattern;
* Repository Pattern;
* UML;
* unit testing;
* integration testing;
* refactoring.

### Частково використано

* async programming;
* advanced patterns;
* performance optimization.

## Готовність релізу

Перед фінальним релізом необхідно:

* успішно пройти всі тести;
* перевірити CI;
* завершити документацію;
* підготувати FINAL_REPORT.md;
* підготувати тег v1.0.0.
