# Future Improvements

## Можливі покращення системи

У майбутньому систему WarehouseManagementSystem можна значно розширити та покращити.

## База даних

Зараз використовується JSON persistence.

У майбутньому можна перейти на:

* SQL Server
* PostgreSQL
* MySQL

Це дозволить:

* швидше працювати з даними;
* виконувати складні запити;
* зберігати великі обсяги інформації.

## Web API

Можна створити ASP.NET Web API для роботи системи через HTTP запити.

Це дозволить:

* підключати frontend;
* створити мобільний застосунок;
* працювати через браузер.

## Frontend

Можна додати:

* ASP.NET MVC;
* Blazor;
* React;
* Angular.

Це зробить систему зручнішою для користувача.

## Авторизація

Можна реалізувати:

* реєстрацію;
* вхід у систему;
* ролі користувачів;
* адміністратора.

## Docker

Можна додати Docker контейнеризацію для:

* швидкого запуску;
* деплою;
* перенесення проєкту між середовищами.

## CI/CD

Можна реалізувати:

* GitHub Actions;
* автоматичне тестування;
* автоматичний build;
* автоматичний deploy.

## Logging

Можна додати:

* Serilog;
* NLog;
* логування помилок;
* логування дій користувача.

## Swagger

Для Web API можна додати Swagger документацію.

## Unit Testing

Можна збільшити:

* покриття тестами;
* кількість integration тестів;
* performance testing.

## Висновок

Проєкт має хорошу основу для подальшого розвитку та може бути розширений до повноцінної enterprise системи.
