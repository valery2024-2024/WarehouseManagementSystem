# Self Audit

## Загальна оцінка проєкту

Проєкт WarehouseManagementSystem реалізований у вигляді багатошарової системи на платформі .NET.

У проєкті реалізовано:

* Domain layer
* Application layer
* Infrastructure layer
* Console UI
* JSON persistence
* Repository pattern
* Strategy pattern
* Unit tests
* Integration tests
* Exception handling
* Документацію

## Що було виконано

Під час виконання лабораторних робіт було реалізовано:

* систему управління товарами;
* систему замовлень;
* систему складу;
* збереження даних у JSON;
* патерн Repository;
* патерн Strategy;
* перевірку бізнес-логіки;
* unit testing;
* integration testing;
* документацію проєкту.

## Сильні сторони проєкту

### Архітектура

Проєкт має зрозумілу структуру та розділення відповідальностей між шарами.

### Тестування

Було створено:

* unit тести;
* integration тести;
* exception тести.

Усі тести успішно проходять через `dotnet test`.

## Що можна покращити

У майбутньому можна додати:

* базу даних SQL Server;
* Web API;
* ASP.NET MVC або React frontend;
* авторизацію користувачів;
* Docker;
* CI/CD;
* логування;
* Swagger документацію.

## Висновок

Проєкт успішно реалізує основні можливості системи управління складом та демонструє використання сучасних підходів до розробки програмного забезпечення на платформі .NET.
